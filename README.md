# ClipKeep

**ClipKeep** — менеджер истории буфера обмена для Astra Linux 1.7 (X11), написанный на Java 17 и JavaFX.

---

## Возможности

- 📋 Автоматическое отслеживание буфера обмена (опрос каждые 500 мс через `xsel`)
- 🗂 Хранение истории в JSON-файле (до 100 записей по умолчанию)
- 📌 Закрепление важных записей (закреплённые записи не удаляются при переполнении)
- 🔍 Мгновенный поиск по тексту
- 🗑 Удаление отдельных записей и очистка всей истории (кроме закреплённых)
- 🖥 Иконка в системном трее с контекстным меню

---

## Требования

| Компонент | Версия |
|-----------|--------|
| Java      | 17+    |
| Maven     | 3.8+   |
| xsel      | любая  |
| Astra Linux / X11 | 1.7+ |

Убедитесь, что `xsel` установлен:

```bash
sudo apt-get install xsel
```

## Сборка

```bash
# Клонируйте репозиторий
git clone <url> clip-keep
cd clip-keep

# Скомпилировать и запустить тесты
mvn verify

# Собрать fat JAR (содержит все зависимости)
mvn package -DskipTests
```

Готовый jar будет находиться в `target/clip-keep-1.0.0-fat.jar`.

---

## Запуск

```bash
java -jar target/clip-keep-1.0.0-fat.jar
```

> **Примечание:** Файл `config.json` должен находиться в текущей директории (той, из которой запускается команда).

### Скрипт-обёртка (опционально)

Создайте файл `/usr/local/bin/clipkeep` для удобного запуска:

```bash
#!/bin/bash
cd /opt/clipkeep
exec java -jar clip-keep-1.0.0-fat.jar "$@"
```

```bash
chmod +x /usr/local/bin/clipkeep
```

## Конфигурация (`config.json`)

```json
{
  "maxHistorySize": 100,
  "pollingIntervalMs": 500,
  "storagePath": "history.json"
}
```

| Параметр           | Описание                                          | По умолчанию     |
|--------------------|---------------------------------------------------|------------------|
| `maxHistorySize`   | Максимальное количество незакреплённых записей    | `100`            |
| `pollingIntervalMs`| Интервал опроса буфера (мс)                       | `500`            |
| `storagePath`      | Путь к файлу истории                              | `history.json`   |

## Архитектура

```
src/main/java/ru/clipkeep/
├── App.java                  # Точка входа (JavaFX-приложение)
├── config/
│   └── AppConfig.java        # Конфигурация (config.json)
├── model/
│   └── ClipItem.java         # Элемент истории (id, text, timestamp, pinned)
├── service/
│   ├── ClipboardService.java  # Низкоуровневый доступ к буферу через xsel
│   ├── ClipboardWatcher.java  # Цикл опроса в фоновом потоке
│   ├── StorageService.java    # Чтение/запись JSON-файла истории
│   └── HistoryService.java    # Бизнес-логика (дубликаты, pin, фильтрация, лимит)
├── tray/
│   └── TrayService.java       # Иконка и контекстное меню в системном трее
└── ui/
    ├── MainWindow.java        # Управление окном JavaFX (Stage)
    └── MainController.java    # FXML-контроллер главного окна
```

```

        │
        └──onNewClip──► HistoryService ──save──► StorageService (JSON)
                               │
                               └──getAll/search──► MainController ──► ListView
```

## Использование

1. При запуске появляется главное окно и иконка в трее.
2. **Закрыть окно** — приложение сворачивается в трей (не завершается).
3. **Открыть снова** — двойной клик по иконке или пункт «Открыть» в меню трея.
4. **Копировать** — клик по записи копирует её в буфер обмена.
5. **Закрепить** — кнопка «Закрепить» / «Открепить» (📌 закреплённые записи не удаляются).
6. **Поиск** — начните вводить в строке поиска, список обновляется мгновенно.
7. **Выход** — пункт «Выход» в контекстном меню иконки трея.

## Тесты

```bash
mvn test
```

Тесты расположены в `src/test/java/ru/clipkeep/service/HistoryServiceTest.java` и проверяют:
- добавление и дедупликацию записей
- соблюдение лимита истории
- защиту закреплённых записей от удаления
- поиск, удаление, очистку
- персистентность (сохранение и перезагрузку с диска)
